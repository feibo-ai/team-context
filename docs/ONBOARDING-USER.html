怎么用这套 AI 工具链

这是 AI MIQ 团队的 AI 工具链:去本地 MCP,改用 Claude Code / Codex 的 skills + multica CLI + 远程 MCP。环境由 agent 配好(见 ONBOARDING-AGENT.html),本文只讲:配好之后,你日常怎么用。

核心工作法 · RPI

Research → Plan → Implement,这是三个分离的 session,不要混在一个里。每个阶段开新 session,用对应的 skill 起头——你说一句话,它自动触发。

怎么触发(说一句话即可)

你要做什么触发 skill怎么说
启动新项目tc-1-start「启动新项目」
调研(R)tc-2-research「调研 X」
写计划(P)tc-3-plan「做个 plan」
写代码(I)tc-4-build「执行 / 实现」
收尾复盘tc-5-review「收尾 / 写 case」
/clear 之前tc-handoff「换个 session / 重开」
卡住自检tc-self-check「自检一下」

multica 约定

  • 每个 issue 必须挂在项目下——建之前先 multica project list 选定项目。
  • 用 multica CLI 管 project / issue(创建、改 label、改 status）。
  • 拿不准是不是这个项目、要不要新建项目——问 DRI,绝不建无项目的孤儿 issue。

几条不可妥协(团队红线)

  1. 写代码前必有 plan 文档 + 第二个 session 评审——不 vibe code。
  2. 每段 AI 生成的 diff,commit 前过人眼——你来 ship。
  3. 每个项目 / 任务以 case 复盘收尾。

卡住怎么办

30 分钟没进展,就触发 tc-handoff(它把状态写进 issue + commit WIP)→ 然后 /clear 重开新 session。别原地耗、别在浑浊的 context 里硬修。

去哪找细节

环境配置 / 安装 → ONBOARDING-AGENT.html(那是 agent 的事,本文不讲)。
团队 SOP、标准、各 skill 详情 → team-context 仓库;某个 skill 怎么用,直接在对话里问它。
