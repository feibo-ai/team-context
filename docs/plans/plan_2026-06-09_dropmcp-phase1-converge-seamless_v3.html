方案

【背景】落实两份 ADR:2026-06-08-drop-local-mcp 与 2026-06-09-rpi-publish-architecture-redeliberation(5-agent 评审 + 3 架构师收敛=不拆库轻量收敛=Phase 1,核心下沉留 Phase 2 条件触发)。北极星(DRI):极致简化安装+使用,分发/更新/采集无感。本 v3 据 3 独立评审 + 闭环再评审对 v1/v2 据实重定价。 【张力的化解 · 据评审修正】无感需分两半看:(一)【遥测/采集】daemon 复用成立——已每 6h 自更 CLI(auto_update,带 claim barrier)、每 1m 采集 transcript 用量 POST(ambient_usage.go:128,counts-only),延伸结构化遥测低成本;(二)【skill 无感分发】daemon 复用不成立——现有 skill 机制只读『成员盘→上报服务端』(local_skills.go 无写盘、daemon.go:1398-1440),要的『registry→静默写可执行 skill』是净新 server 分发 + 净新 daemon 写路径,且 skill 含可执行脚本=RCE 面。故无感分发是【结构性】;DRI 拍板据实纳入(把一小片 Phase2 daemon-化提前)。 【评审记录 · v1→v2→v3】v1:3 独立子agent(架构/安全/SOP)一致 changes-requested,对照代码证伪三前提(①无感分发非低成本延伸→实为净新 server+daemon写+RCE;②自更非无 checksum→SHA-256 已 fail-closed update.go:178,真缺签名;③漏:自更 self-host 默认关 config.go:381)+ buildSkillMd 实只缺 owner(:267 last_reviewed_at 已写、owner 关系字段 :540)+ ②④①三 buried 子项未进可观测标准。v2 全改并升标准。v2→v3:闭环再评审 8/10 闭合,揪出剩 2 项——(2)签名未定信任根:若签名方=推送方则校验循环、零防护;(2b)自相矛盾:标准⑩静默落盘 vs 安全段『可执行 skill 走确认』,且『内容 vs 可执行』判别不可行(团队 skill 皆带脚本、静默桶近空)。v3 据 DRI『全员有 git 权限』定解:信任根=team-context git 成员签名集(独立于 registry/server 推送 → 非循环;server 被攻破伪造不了成员签名),验签过才静默落盘、失败拒绝+告警,人在环前移到『签名提交评审』一次(SOP 每段 diff Schema + 阈值派生 + 折叠空白;③ 删 tc-handoff 绕过口;④ buildSkillMd thread owner_user_id + sync-team-config create-or-update+files[] + CI 对账;⑤ multica doctor preflight;⑥ Codex 对称+skills-index;⑦ red-team 四项。 【无感地基
