# Monday Kickoff — 30-min protocol

## Setup (autopilot does this)
By 09:30, `autopilots/monday-kickoff.yaml` posted to feishu:
- All issues with label `计划-已批准` from this week
- For each: title + DRI + appetite + last "Current State" handoff (if any)

## The 30-min protocol (DRI runs)

### Minute 0-5 — Reading time (silent)
Everyone reads the feishu roundup. Don't talk yet.

### Minute 5-20 — DRI walks through plans
DRI calls each plan: "X is doing P. Anyone need to know more?"
NOT: rehearsing plan content. Plans are READ MATERIAL, not presentation.

### Minute 20-25 — Cross-plan boundaries
DRI: "Where do these projects touch each other? Any conflicts?"
The ONLY value the meeting adds beyond solo reading.

### Minute 25-30 — Open floor + close
3-5 min for "anything else." Then end.

## What this meeting does NOT do
- ❌ Daily standup format (dead — see SOP P-6 Daily)
- ❌ Rehearse plan content (plans are already written)
- ❌ Status reporting (autopilot daily summaries cover that)
- ❌ Brainstorm new projects (those go through phase-01-kickoff)

## End-state
By 09:55, everyone knows what others work on this week, where boundaries are, who to ping for collaboration. No notes needed — it's all in the plans.
